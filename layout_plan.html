00 / THE PROBLEM
What's wrong with the current layout?
The current design puts the animation first and the code second — like a theatre where the stage takes 60% of the space and the script is crammed in a side pocket. For a learning platform the priority should be reversed. Learners need theory, pseudocode, and code editor front and centre, with visualization as an on-demand tool.

◆ KEY DESIGN DECISIONS
VIZ PANEL DEFAULTS
Expanded by default on first load, so new users see the animation. Collapsed state remembered per-user via localStorage for return visits.
VIZ PANEL WIDTH
320px expanded (enough for bars + tree), 40px collapsed (just a toggle strip). Resizable via drag handle in future.
PSEUDOCODE SYNC
When the viz panel steps forward, the active pseudocode line highlights in sync — the two panels talk to each other via shared frame state in Zustand.
KEYBOARD SHORTCUT
V toggles the visualizer panel. Space still plays/pauses.
03 / CONTENT TABS
What lives in each tab
The three-tab layout lets users enter the content at the depth they need. A beginner starts with LEARN, a student revising starts with PSEUDOCODE, a coder starts at CODE EDITOR.
◈ TAB 1 — LEARN
  • Algorithm name + one-line definition
  • Plain-English step-by-step explanation
  • Time & space complexity cards with Big-O notation
  • Best / average / worst case breakdown
  • When to use this algorithm (use cases)
  • Comparison with similar algorithms
  • Real-world examples (e.g., "used in Python's sort()")
  • Inline pseudocode snippet preview
  • ▶ "Try it in the visualizer" CTA button
{ } TAB 2 — PSEUDOCODE
  • Full pseudocode in a readable step format
  • Line-by-line highlighting synced with viz panel
  • Collapsible sub-steps for complex algorithms
  • Annotation notes on hover (explains the why)
  • Toggle between pseudocode and flowchart view
  • Step counter tied to viz playback position
  • Copy pseudocode to clipboard button
⌨ TAB 3 — CODE EDITOR
  • Full Monaco editor (expanded, no cramped 380px)
  • Starter code pre-loaded for each algorithm
  • Run button — executes on backend, returns frames
  • Active line synced to viz panel when running
  • Output console for stdout / stderr / exec time
  • Language selector (Python default, JS/Java future)
  • Reset to starter code button
  • Custom input array field
04 / AD STRATEGY
Ad placements that don't break flow
Four dedicated ad zones, each placed at natural visual breaks so they feel integrated rather than intrusive. No mid-content interruptions. No pop-overs. All standard IAB sizes for easy Google AdSense / programmatic integration.
SLOT A — TOP LEADERBOARD
Top Banner
728×90 · Leaderboard
Sits between the navbar and the algorithm info bar. High visibility, seen on every page load. Standard leaderboard — best CPM. Does not scroll with the page.
LEADERBOARD — 728×90
SLOT B — SIDEBAR RECTANGLE
Sidebar Medium Rectangle
300×250 · Medium Rectangle
Placed at the bottom of the left sidebar algorithm list. Sticky within the sidebar — stays visible while user scrolls content. 300×250 is the highest-fill IAB format globally.
MREC — 300×250
SLOT C — IN-CONTENT STRIP
In-Content Banner
728×90 · Leaderboard
Appears between the last content section and the page footer in the main learning column. Natural reading pause — highest viewability score for in-content placements. Not shown mid-paragraph.
IN-CONTENT BANNER — 728×90
SLOT D — COLLAPSED VIZ PANEL
Viz-Collapsed Rectangle
160×600 · Wide Skyscraper
When the visualizer panel is collapsed, the freed right-column space (320px) can optionally show a wide skyscraper or half-page ad. Only shown when user has collapsed the panel — not forced. Premium placement.
SKYSCRAPER 160×600
05 / VIZ PANEL BEHAVIOUR
Three states of the visualization panel
The panel has three distinct modes — collapsed strip, expanded visualizer, and fullscreen overlay. Each is reachable with one click or keypress.
STATE 1 — COLLAPSED STRIP
▶
VISUALIZE
40px strip on the right edge. Shows a ▶ VISUALIZE button and label. Content column takes full remaining width. Ad slot D can appear here. Triggered by: clicking ◀ collapse, pressing V.
STATE 2 — EXPANDED (320px)
VIZ
◀
Default state. 320px right panel showing animation canvas, step description, progress bar, and playback controls. Resizable to 480px via drag handle in future iteration.
STATE 3 — FULLSCREEN OVERLAY
◆ FULLSCREEN VISUALIZER — BUBBLE SORT
✕ EXIT
Press F or click the expand icon to go fullscreen. The visualizer takes over the entire content area. ESC or ✕ returns to expanded state. Best for demos and presentations.
06 / IMPLEMENTATION PLAN
Build it in 3 phases
Each phase is independently shippable and doesn't break the existing experience. Phase 1 alone gives you the new layout. Phases 2 and 3 build on top.
PHASE
01
LAYOUT RESTRUCTURE
Rebuild the VisualizerPage layout. Move Monaco editor to the center column. Make the visualization panel collapsible with a toggle button and keyboard shortcut. Add the three-tab structure (Learn / Pseudocode / Code Editor) to the center column. Wire up the Zustand store for panel open/closed state with localStorage persistence.
VisualizerPage.tsx Zustand store CSS layout overhaul Collapsible panel ~2–3 days
PHASE
02
CONTENT SYSTEM
Populate the Learn and Pseudocode tabs with real content. Extend the algorithmRegistry.ts to include description text, use-case notes, and pseudocode arrays per algorithm. Build the pseudocode viewer component with line-highlighting synced to the current viz frame. Revive the pseudocode panel that currently exists as a component but isn't rendered.
algorithmRegistry.ts PseudocodePanel.tsx LearnTab.tsx Content for 12 algorithms ~3–4 days
PHASE
03
AD INTEGRATION + POLISH
Add the four ad slot placeholder components (AdSlot.tsx) with defined IAB dimensions. Integrate Google AdSense or placeholder divs with correct data attributes so they're ready for programmatic fill. Add fullscreen mode to the viz panel. Add drag-to-resize handle on the viz panel border. Responsive breakpoints for <1200px.
AdSlot.tsx Google AdSense setup Fullscreen mode Drag-to-resize panel ~2–3 days
